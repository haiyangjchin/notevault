# 长时间运行代理设计

## 核心挑战

长时间运行代理的核心挑战是：**它们必须在离散的会话中工作，每个新会话从零开始**。

想象一下一个软件项目由轮班工程师工作，每个新工程师对上一班发生的事情毫无记忆。由于上下文窗口有限，且大多数复杂项目无法在单个窗口内完成，代理需要一种方式来桥接编码会话之间的差距。

## 失败模式

根据 [Anthropic 的研究](https://www.anthropic.com/engineering/effective-harnesses-for-long-running-agents)，长时间运行代理常见两种失败模式：

### 1. 一次完成（One-shotting）
代理试图一次性完成所有工作，导致：
- 上下文耗尽
- 功能半实现
- 文档缺失
- 下一个会话需要猜测发生了什么

### 2. 过早宣布完成
代理看到已有一些进展，就宣布工作完成，即使还有很多功能未实现。

## 解决方案：两阶段架构

### 阶段 1：初始化代理

初始化代理在首次运行时设置环境，包括：

#### 功能列表
- 将用户提示分解为具体功能
- 每个功能都有明确的测试步骤
- 初始状态标记为"失败"

```json
{
  "category": "functional",
  "description": "新建聊天按钮创建新对话",
  "steps": [
    "导航到主界面",
    "点击'新建聊天'按钮",
    "验证新对话已创建",
    "检查聊天区域显示欢迎状态",
    "验证对话出现在侧边栏"
  ],
  "passes": false
}
```

#### 进度文件
- `claude-progress.txt`：记录代理的工作历史
- Git 提交：版本控制所有更改

#### 初始化脚本
- `init.sh`：启动开发服务器
- 基础测试：验证应用状态

### 阶段 2：编码代理

编码代理在每个后续会话中：

#### 1. 获取状态
```bash
# 查看当前目录
pwd

# 读取进度文件
cat claude-progress.txt

# 读取功能列表
cat feature_list.json

# 查看 Git 日志
git log --oneline -20
```

#### 2. 验证基础功能
```bash
# 运行初始化脚本
./init.sh

# 测试基本功能
# 使用浏览器自动化验证
```

#### 3. 增量式工作
- **一次只做一个功能**
- 完成后提交 Git
- 更新进度文件

#### 4. 留下清晰的交接
- 描述性提交消息
- 详细的进度更新
- 干净的代码状态

## 关键实践

### 1. 增量式进展
> "这种方法将问题分解为两部分：首先，设置初始环境；其次，提示每个代理做出增量进展。"

**为什么重要**：
- 防止代理一次做太多
- 确保每个功能都经过测试
- 便于下一个会话继续

### 2. 测试驱动
> "代理倾向于标记功能为完成，而没有适当测试。"

**解决方案**：
- 使用浏览器自动化（Puppeteer MCP）
- 像真实用户一样测试
- 只有在测试通过后才标记为"通过"

### 3. 环境管理
- **功能列表**：JSON 格式，防止代理随意修改
- **进度文件**：记录所有工作历史
- **Git**：版本控制和回滚能力
- **初始化脚本**：快速启动开发环境

## 上下文重置 vs 压缩

### 上下文压缩（Compaction）
- 总结早期对话内容
- 保持连续性
- 但不给代理"干净的开始"

### 上下文重置（Context Reset）
- 完全清除上下文窗口
- 开始全新的代理
- 通过结构化交接传递状态

**选择依据**：
- 如果模型表现出"上下文焦虑"（过早结束工作）→ 使用重置
- 如果模型能很好地处理长上下文 → 使用压缩

## 实际案例

### Claude.ai 克隆项目

**提示**："构建 claude.ai 的克隆"

**初始化代理设置**：
- 200+ 个功能
- 功能列表（JSON 格式）
- 进度文件
- 初始化脚本

**编码代理工作流程**：
1. 读取状态
2. 选择下一个功能
3. 实现功能
4. 测试功能
5. 提交更改
6. 更新进度

**结果**：成功构建了一个功能完整的 Web 应用

## 最佳实践总结

| 问题 | 初始化代理行为 | 编码代理行为 |
|------|---------------|-------------|
| 代理过早宣布完成 | 设置功能列表文件 | 读取功能列表，一次只做一个功能 |
| 代理留下有 bug 的代码 | 初始 Git 仓库和进度文件 | 读取进度和 Git 日志，运行基础测试 |
| 代理过早标记功能完成 | 设置功能列表文件 | 自我验证所有功能，测试通过后才标记 |
| 代理花时间弄清楚如何运行应用 | 编写 `init.sh` 脚本 | 读取 `init.sh` |
